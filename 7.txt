3)
✅ Step 1: Create a Kubernetes YAML file

Create a file called resume-deploy.yaml:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: resume-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: resume-app
  template:
    metadata:
      labels:
        app: resume-app
    spec:
      containers:
        - name: resume-container
          image: resume-app       # or your-docker-username/resume-app
          ports:
            - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: resume-service
spec:
  type: NodePort
  selector:
    app: resume-app
  ports:
    - port: 80
      targetPort: 80
      nodePort: 30080


In exams, this single file (Deployment + Service) is perfect.

✅ Step 2: Apply the YAML

In the folder where resume-deploy.yaml is:

kubectl apply -f resume-deploy.yaml

✅ Step 3: Verify
kubectl get pods
kubectl get svc resume-service


You should see:

A pod from resume-deployment

A service resume-service with NodePort 30080

✅ Step 4: Access in browser

If you’re using a local cluster (like Docker Desktop or Minikube):

http://localhost:30080
